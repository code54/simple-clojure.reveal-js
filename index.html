Sintaxis y Evaluación

Todo es evaluación de Expresiones

No existen los "statements"

Datos atómicos


       true false        ;; booleans

       0 123 3.14  22/3  ;; Numbers

       "JavaConf 2014"   ;; Strings

       \a \z             ;; Chars

       user-name         ;; Symbols

       :key1             ;; keywords

Collecciones


           [1 2 "tres"]      ;; Array
           
           (1 2 "tres")      ;; List


           {:key1 "value1"
            :key2 "value2" } ;; Map


           #{ 1 2 3 4 5 }  ;; Set

Evaluación

Todas las expresiones anteriores se evaluan a sí mismas.

Exceptuando: symbols y listas

Symbols

Nombres que hacen referencia a otro dato

Ejemplo: El nombre de un valor o una función:


            +
            concat
            my-number
            my-function

Listas

Son evaluadas como operaciones

       (operacion arg1 arg2 arg3)

Operación: una función, un macro o una "forma especial"

Operaciones: Funciones

     (function-name arg1 arg2 arg3)

Eager evaluation: Primero se evalua cada uno de los argumentos, luego se aplica la función


       (println "Hello " user-name)

             (+ 1 2 3 4 5)

         (concat [1 2 3] col2 col3)

"Special Forms"

Primitivas provistas por el lenguaje

Evaluación de argumentos es particular para cada special form

    (if condition-exp true-exp false-exp)

   (if (> danger-prob 0.9)

Macros

Son funciones que ejecutan en tiempo de evaluación/compilación

Transforman expresiones en nuevas expresiones

Son una forma de extender el languaje

Todo el poder del lenguage está disponible

Muchas operaciones de la librería estándar son macros
